# Materialised into a per-plugin venv at `plugin_install` time (ADR 0064
# §3.11) -- never at spawn, so the registered MCP servers start network-free.
#
#   - sqlite-vec (dual MIT/Apache-2.0): pure-C SQLite extension, no bundled
#     embedding model -> no HuggingFace-fetch hazard (FP-0057 line 55).

#     interpreter builds). apsw statically bundles its own SQLite with
#     extension loading always enabled, so loading is portable.
#   - chonkie (MIT): recursive/semantic/etc. chunking, ~15 MiB vs LangChain's
#     ~80 MiB; no bundled model in the default character-tokenizer path used
#     here -> no network fetch at import or call time.
#   - fastmcp: the stdio MCP server framework both scripts build on.
