# 巻き戻しと分岐の方法

Reyn のタイムトラベルは、過去のチェックポイントへ巻き戻し、必要に応じてその時点から分岐できる機能です。本ガイドでは利用可能なコマンドと UI を説明します。

内部の仕組みについては [タイムトラベルのコンセプト](../../concepts/runtime/time-travel.md) を参照してください。

## Rewind ピッカーを開く

```
/rewind
```

TUI 上でインタラクティブなチェックポイントのタイムラインが開きます。各行には以下が表示されます:

| カラム | 説明 |
|--------|------|
| seq | チェックポイントのグローバルシーケンス番号 |
| time | チェックポイントが作成されたタイムスタンプ |
| kind | 境界の種類: `turn` / `plan-step` / `phase` |

**↑ / ↓** で移動し、**Enter** で選択します。**Esc** を押すと巻き戻さずに閉じます。

**ショートカット**: TUI のどこからでも **Esc Esc** をダブルタップするとピッカーを直接開けます。

## 特定の seq へ巻き戻す

```
/rewind <N>
```

ピッカーを開かずに seq N へ直接巻き戻します。会話の状態とワークスペースのファイルの両方が、seq N 時点の状態に復元されます。

## 分岐ツリーを移動する

巻き戻しまたは fork のあと、ピッカーは全分岐を表示する **ツリービュー** に切り替わります。各分岐はそのアンカーとなるチェックポイントでラベル付けされます。↑ / ↓ + Enter で、アクティブ・破棄済みを問わず任意の分岐上のチェックポイントを選択できます。

- **現在の分岐** 上の seq を選択: undo（現在の分岐を巻き戻す）。
- **非アクティブな分岐** 上の seq を選択: fork-switch（その分岐をアクティブ化する）。

## Web 編集 (Phase 2d)

Reyn を Web インターフェース（WebSocket / A2A）経由で利用する場合、`/rewind` は同じチェックポイントピッカーを開きます。分岐元のチェックポイントを選択すると、Web 編集フローは元のメッセージを提示し、編集版を再入力して送信できます。Web 側ではインラインのプリフィルに対応していないため、置き換えるテキストを直接入力します。送信すると、巻き戻したチェックポイントから新しい fork が作成されます。

## コンテナモード

コンテナ環境バックエンド（`--container`）を使う場合、ワークスペースの巻き戻しは shadow-git の `as-of-N` を介してコンテナのファイルシステム上で動作します。ユーザー体験は同じで、基盤がコンテナ側になります。

## タイムトラベルのコストを調整する

タイムトラベルはデフォルトで有効です。最大の固定コストは、各チェックポイント境界（毎ターン・毎 plan-step）で実行される **ワークスペースの shadow-git キャプチャ** で、これにより巻き戻し時にリポジトリのファイルを復元できます。これは `reyn.yaml` の `time_travel` ブロックで調整できます。キーごとの完全なリファレンスは [`reyn.yaml` § time_travel](../../reference/config/reyn-yaml.md#time_travel-block)、なぜそのコストがかかるのかは [コストとランタイム限定のオプトアウト](../../concepts/runtime/time-travel.md#cost-and-the-runtime-only-opt-out) を参照してください。

### ファイル巻き戻しをオプトアウトする (`workspace_capture: false`)

```yaml
# reyn.yaml
time_travel:
  workspace_capture: false   # デフォルトは true
```

これは **ランタイム限定の巻き戻し** を選択します。`/rewind`、fork ピッカー、分岐、checkout はすべて動作し続け、**エージェント + 会話の状態** を復元しますが、**作業ツリーは変更されません**（リポジトリのファイルはスナップショットも巻き戻しもされません）。

有効なケース:

- ワークスペースが **大きい** — 境界ごとの `git add -A` がツリー全体を stat する;
- **コンテナ** で実行している — 各キャプチャが `docker exec` のラウンドトリップになる;
- 巻き戻しは常に **過去の時点から再実行する** ためだけに使い、その時点の *当時のリポジトリ* を検査するためには使わない。

これは act-turn 巻き戻しがすでに提供しているのと同じ忠実度です（「ここから再実行」であって「ここでの当時のリポジトリ」ではない）。クラッシュリカバリには **影響しません** — リカバリはワークスペースキャプチャではなく WAL に乗っています。この設定は **起動時に読み込まれ**（run レベル）、セッション途中のトグルではありません。

### per-step キャプチャをオプトインする (`act_turn_capture: true`) — 上級

```yaml
# reyn.yaml
time_travel:
  act_turn_capture: true   # デフォルトは false
```

これはより安価な **op ごと** のワークスペーススナップショット（ターン / plan-step ごとではなく skill-run のステップごとに 1 回）を追加します。**デフォルトで無効** かつ **高頻度**（op ごとに 1 キャプチャ）です。

**正直なステータス**: 現時点でこれを有効にしても **ユーザーから見える巻き戻しのメリットはありません** — ピッカーはまだ skill-run の *途中* に巻き戻しを着地させる手段を公開していないため、追加の per-step スナップショットは将来の機能に向けた基盤であり、現在ナビゲートできるものではありません。その基盤を前もってキャプチャしておきたい特別な理由がない限り、無効のままにしてください。また `workspace_capture: false` のときは **no-op** です（per-step キャプチャは同じワークスペースストアに乗っています）。
