# reyn

**LLM エージェントのための OS ── エージェントが判断・組織化・オーケストレーションを担い、OS はあらゆる操作を型付け・権限管理・監査・復旧可能に構造的に保証する。**

reyn では LLM エージェントが判断し、組織化し、オーケストレーションします ── ツールを選び、サブエージェントを起動し、pipeline を組み立てます。その一方で OS はあらゆる操作を型付け(Control IR)・権限ゲート済み・監査ログ済み・クラッシュから復旧可能な状態に保ちます。エージェントの自由度は本物であり、OS の保証はエージェントの規律に頼るものではなく構造的なものです。

## Powered by AI

reyn は 2 つの意味で「 AI 駆動」 です:

- **ランタイム**: すべてのエージェント実行は LiteLLM 経由で LLM プロバイダに判断を委譲します。reyn は、判断・起動・オーケストレーションが型付け・権限管理・監査・巻き戻し可能な op のみを通して行われるよう、エージェントの自由度を構造的に制約する agent OS です。
- **開発**: コードベース、 stdlib スキル、 本ドキュメント、 ランディングページの相当部分は AI ツール (= 主に Claude Code (Anthropic) による実装、 Claude Design による Web サイト) で起草されました。 人間によるレビュー / 統合 / 最終的なアーキテクチャ判断はメンテナが担い、 AI の寄与は git 履歴の `Co-Authored-By: Claude ...` trailer として記録されています。

これはマーケティングコピーではなく透明性のための開示です。 来歴を監査する場合は `git log --grep="Co-Authored-By: Claude"` および `website/_design/` 配下のデザインプロンプトを参照してください。
